# Daily Summary — 2026-06-29

## Key highlights
- **DSpark v7 released with prefix cache support** — Festr published the v7 Docker image with DeepSpark (dspark) for DS4 Flash, achieving 260–320 tok/s at `--tp 2` with prefix cache working. The PR is live at vllm upstream. [(jump)](https://discord.com/channels/1466898002793857221/1481304034198421515/1520877151405084873)
- **DSpark benchmarks vs MTP=2**: ormandj's real-world test (171k context, agentic Rust coding) shows DSpark at ~216 tok/s decode vs MTP n=2 at ~187 tok/s (+15%), with DSpark yielding ~1.6× more tokens per forward pass (60% acceptance, 3.98 tok/step vs 75% at 2.49). [(jump)](https://discord.com/channels/1466898002793857221/1481304034198421515/1521019461614702612)
- **TP=4 DSpark crash bug identified** — `AssertionError: warp_col_tiles must be greater than 8, got 4`. Claude-suggested fix: repartition warps (one-line change in `dspark_sparse_attn_tilelang.py`). TP=4 brandonmusic benchmarks show 235.6 decode / 13,516 prefill 8k vs 182.6/9,074 at TP=2. [(jump)](https://discord.com/channels/1466898002793857221/1481304034198421515/1520851731280302131)
- **vLLM config insight**: KV cache printout during startup is an approximation with DS attention — setting `max-model-len 1M` bumps reported size but physical slots are unchanged. ormandj's tuned config achieves 1.1M KV cache tokens at TP=2. [(jump)](https://discord.com/channels/1466898002793857221/1481304034198421515/1520969806675705877)
- **GLM-5.2 TP=6 KV head padding bug fixed** — Padding inflated to 96 heads (should be 66) on TP=6, eating VRAM. Workaround: set `_ATTENTION_HEAD_LOCAL_ALIGNMENT = 1`. `FLASHINFER_MLA_SPARSE_SM120` now officially merged upstream; DCP support still in fork. [(jump)](https://discord.com/channels/1466898002793857221/1471527895439638528/1520910362386632785)
- **GLM-5.2 on 4x DGX Spark cluster**: 27 tok/s decode (non-pruned) reported in NVIDIA forums thread. [(jump)](https://discord.com/channels/1466898002793857221/1517199740444475423/1520787852877631499)
- **C-Payne switch PCIe issue**: koush diagnosed ~50% prefill throughput regression (1638→819 tok/s oscillating) via AER errors on passive MCIO adapters at PCIe 5.0; stable at 4.0. PCIe 5.0 requires retimers. [(jump)](https://discord.com/channels/1466898002793857221/1474436297878933699/1521037524061913248)
- **MiniMax M3 NCCL tuning**: Storm found forcing Ring+Simple with 1 channel improved TP=4 all-reduce from 17.1→21.0 GiB/s (~18% gain), raising 12-agent throughput from ~54→63 gen tok/s. [(jump)](https://discord.com/channels/1466898002793857221/1514993479359529120/1520975050705076324)
